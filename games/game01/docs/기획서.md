# 직장인 잔혹사 - 게임 기획서

## 개요

- **장르**: 미니게임 컬렉션 (모바일 최적화)
- **화면**: 세로(Portrait) 모드 / 390×844 (9:19.5, 아이폰 14/15 기준)
- **플랫폼**: 모바일 웹
- **기술 스택**: React + Phaser 3 + TypeScript

## 게임 구조

총 **5개 스테이지**로 구성된 직장인의 하루를 체험합니다.

| # | 카테고리 | 시간 | 게임 | 장르 |
|---|----------|------|------|------|
| 1 | 출근 | 07:00 AM | 천국의 계단 | 무한의 계단 |
| 2 | 오전 | 09:00 AM | 보고서 오탈자 | 컬링 |
| 3 | 점심 | 12:00 PM | 메뉴 받아내기 | 기억력+캐치 |
| 4 | 오후 회의 | 14:00 PM | 회의록 작성 | 테트리스 |
| 5 | 퇴근 | 18:00 PM | 술 버리기 | 타이밍 액션 |

## 스테이지 상세

### STAGE 1. 출근 — 천국의 계단

> "아 늦었다! 출근!!"

- **장르**: 무한의 계단
- **컨셉**: 지각 직전! 계단을 올라 회사에 도착하라!

#### 조작법 (하단 2버튼)
- **왼쪽 버튼** (🔄 방향전환): 캐릭터가 바라보는 방향을 좌↔우 전환
- **오른쪽 버튼** (⬆ 올라가기): 현재 바라보는 방향으로 계단 한 칸 올라감

#### 게임플레이
- 계단이 2~5칸씩 같은 방향으로 이어지다가 꺾임
- 캐릭터 방향과 다음 계단 방향이 맞으면 올라감
- 방향이 틀리면 **추락 + 3초 시간 패널티** (게임 오버 아님)
- 캐릭터는 항상 **화면 중앙에 고정**, 계단이 X/Y 모두 스크롤
- 캐릭터 위에 **➡/⬅ 큰 화살표**로 현재 방향 표시 (전환 시 펄스 애니메이션)
- 10칸마다 콤보 팝업

#### 이미지 교체
텍스처 키를 `preload()`에서 등록하면 플레이스홀더 대신 사용됨:
- `'stair'` : 계단 한 칸 (60×32)
- `'player-idle'` : 캐릭터 서있기 (30×44)
- `'player-step'` : 캐릭터 걷기
- `'player-fall'` : 캐릭터 추락

#### 점수 시스템
| 이벤트 | 점수 |
|--------|------|
| 계단 1칸 올라감 | +1 |
| 추락 | 시간 -3초 |
- **점수** = 올라간 계단 수 (상한 없음)

### STAGE 2. 오전 — 보고서 오탈자 (컬링)

> "막내야, 보고서 오탈자 좀 확인해봐"

- **장르**: 퍼즐보블 스타일 지우개 발사
- **컨셉**: 보고서의 오탈자를 직접 읽고 찾아서 지우개로 수정! 오탈자는 표시되지 않음 — 플레이어가 눈으로 찾아야 함

#### 레이아웃
- **보고서 영역** (y=80~550): 흰색 A4 패널, 10줄 빽빽한 한글 보고서 (제목/작성자/본문/끝)
- **캐릭터** (하단 중앙 고정): 파란 사각형 캐릭터 (퍼즐보블 발사대 역할)
- **조준선**: 캐릭터에서 뻗어나가는 빨간 점선 + 끝점 원
- **파워 게이지**: 캐릭터 위에 표시 (홀드 시 차오름, 초록→빨강)

#### 조작법 (퍼즐보블 스타일, 2단계 분리)
1. **드래그 (좌우)**: 조준선 각도 조절 (12px 이상 이동 시 조준 모드, 손 떼면 각도 확정)
2. **꾹 누르기 (가만히 홀드)**: 파워 게이지 충전 (최대 2초) → 손 떼면 발사

#### 지우개 물리
- 조준 방향으로 비행 (직선, Cubic.easeOut 감속 — 컬링 느낌)
- 파워에 따라 이동 거리 결정: `80 + power² × 420` px (짧게 누르면 가까이)
- 비행 시간: `600 ~ 1100ms` (파워 낮을수록 느림)
- 착탄 위치에서 50px 이내 가장 가까운 단어와 충돌 판정

#### 보고서 데이터
- 3세트 중 랜덤 선택, 10줄 한글 보고서 (제목/작성자/본문 형식)
- 각 줄 1개 오탈자 (총 ~10개)
- **오탈자 표시 없음** — 모든 글자가 동일한 검정색, 밑줄 없음
- 오탈자 예시: "마케팅"→"마게팅", "검토"→"검트", "결재"→"결제", "공유"→"공류"

#### 점수 시스템
| 이벤트 | 점수 |
|--------|------|
| 오탈자 수정 성공 | +100 |
| 정상 문구 삭제 (미스) | -50 |
| 전체 오탈자 수정 클리어 보너스 | +500 |
| 남은 시간 보너스 (클리어 시) | timeLeft × 10 |

#### 비주얼
- 보고서: 흰색 배경 패널, 14px 검정 텍스트 (오탈자 구분 없음)
- 수정 성공: 해당 단어 초록색으로 변경 + "수정 완료! +100" 팝업
- 수정 실패: 해당 단어 회색 처리 + "앗! 멀쩡한 글씨가... -50" 빨간 팝업 (흔들림)
- 지우개: 분홍색 사각형 + 🧽 이모지

### STAGE 3. 점심 — 메뉴 받아내기

> "와 점심시간! 직원들이 각자 주문을 얘기해!"

- **장르**: 기억력 + 캐치 게임
- **컨셉**: 직원들이 외치는 주문을 **외우고**, 올바른 메뉴만 골라 받아내라!
- **조작법**: 드래그로 좌우 이동 (바구니)

#### Phase 1: 주문 암기
- 8명의 캐릭터가 그룹(1→2→3→2)으로 순서대로 등장
- 각 캐릭터 위에 말풍선으로 "나는 치킨!" 표시
- 그룹당 ~2초 보여준 뒤 다음 그룹으로 슬라이드 전환
- 마지막에 전체 주문 요약을 1.5초간 보여준 후 "시작!" → Phase 2 전환

#### Phase 2: 바구니 캐치
- **정답 풀**: 못 받은 메뉴들이 항상 풀에 남아있음
- **떨어지는 아이템**: 정답 풀에서 랜덤 1개 vs 오답 중 확률적으로 선택
- 정답을 캐치하면 풀에서 제거, 풀이 비면 클리어
- **캐치 중 정답 목록 미표시** — 순수 기억력으로 플레이
- 낙하 속도 150px/s, 생성 간격 1.2초

#### 오답 메뉴 시스템
- 전체 아이템의 ~45%가 오답
- 오답 구성 (50:50):
  - **헷갈리는 변형**: 주문 메뉴와 비슷한 이름 (치킨→치킹, 짜장면→자장면, 불고기→볼고기 등)
  - **완전 다른 것**: 주문에 없는 음식 (콜라, 탕수육, 군만두 등)
- 모든 아이템은 동일한 외관 — 플레이어가 기억으로 구분해야 함

#### 메뉴 풀
```
김치찌개, 된장찌개, 비빔밥, 불고기, 떡볶이,
삼겹살, 냉면, 짜장면, 짬뽕, 칼국수,
김밥, 라면, 제육볶음, 순두부, 갈비탕,
돈까스, 치킨, 피자, 햄버거, 초밥
```
- 매 판 풀에서 랜덤 8개 선택 (같은 메뉴 최대 2개까지 중복 허용)

#### 점수 시스템
| 이벤트 | 점수 |
|--------|------|
| 정답 메뉴 캐치 | +10 |
| 오답 메뉴 캐치 (주문 안 한 것 / 헷갈리는 변형) | -10 |
| 초과 캐치 (이미 다 받은 메뉴 또 받기) | -10 |
| 클리어 보너스 (8개 전부 정답) | +50 + 빠른 배달 보너스 |
| 놓침 (정답이든 오답이든) | 0 (패널티 없음) |

#### 클리어 조건
- 8개 주문 메뉴를 모두 정확히 받으면 → **주문 완료!** + 보너스 점수 → 다음 스테이지로
- 빠른 배달 보너스: 30초 기준, 빨리 끝낼수록 시간 보너스 증가
- 60초 전체 타이머 — 시간 내 못 끝내면 현재 점수로 마감

### STAGE 4. 오후 회의 — 회의록 작성 (테트리스)

> "팀장: 회의 할께요! 막내야 받아적어!"

- **장르**: 테트리스 (아래에서 올라오는 변형)
- **컨셉**: 회의록 작성을 테트리스로! 아래에서 기획서 본문이 한 글자씩 적힌 블록이 올라오고, 위에서 테트로미노를 떨어뜨려 빈 칸을 채워 줄을 깨면 회의록 완성!

#### 그리드
- **12열 × 24행**, 셀 크기 28px
- 그리드 너비 336px (390px 화면에서 좌우 마진)
- 배경: 어두운 회의실 톤 (`#1a1a2e`), 격자선 (`#2a2a3e`, 50% 투명)

#### 조작법 (드래그 기반, 축 고정)
- **좌우 드래그** → 피스 좌우 이동 (셀 너비 28px 단위로 1칸씩)
- **아래로 드래그** (50px 이상) → 하드드롭 (즉시 착지)
- **탭** (드래그 없이 터치 후 떼기) → 회전
- **축 고정 시스템**: 터치 후 12px 이상 움직이면 가로/세로 방향 판별 → 한 번 결정되면 해당 축으로만 동작 (아래로 내리려다 좌우로 밀리는 오입력 방지)

#### 게임플레이

##### 아래에서 올라오는 라인 (기획서 본문)
- 게임 시작 시 **1줄**이 미리 올라옴
- 이후 일정 간격으로 **1줄씩** 아래에서 추가 (기존 블록 전체가 1칸씩 위로 밀림)
- 각 줄은 12칸 중 **10~11칸**이 회색 블록(`#5a5a6e`)으로 채워지고 **1~2칸은 빈칸**
- 채워진 블록에 **기획서 본문이 한 글자씩 순서대로** 표시
  - 왼쪽→오른쪽, 아래→위로 자연스럽게 읽히도록 순차 배치
  - 본문: "금일 회의 안건을 공유드립니다. 1분기 매출 목표 달성률 검토 및 2분기 마케팅 전략 수립이 필요합니다..." (회의 관련 문서)
  - 본문이 끝나면 처음부터 반복
- **블록이 맨 윗줄(0행)에 도달한 상태에서 새 줄 추가 시 → 게임 오버**

##### 플레이어 테트로미노 조작
- 위에서 표준 테트로미노 7종 (I, O, T, S, Z, J, L) 랜덤 낙하
- 각 피스별 고유 컬러: I=시안, O=노랑, T=보라, S=초록, Z=빨강, J=파랑, L=주황
- **고스트 피스**: 착지 예상 위치에 25% 투명도로 표시
- **회전**: 시계방향 90° 회전 + 월킥 (오프셋 0, ±1, ±2 시도)
- **자동 낙하**: 일정 간격으로 1칸씩 하강
- **하드드롭**: 즉시 최하단까지 낙하 후 고정

##### 라인 클리어
- 가로줄 12칸이 전부 채워지면 해당 줄 제거 → 위 블록 아래로 떨어짐
- **회의록 줄** (글자가 있는 줄) 클리어 시 → "회의록 작성 완료! +점수" 팝업
- **일반 줄** (글자 없는 줄) 클리어 시 → "줄 클리어! +점수" 팝업
- **다중 클리어** 시 → "N줄 클리어! +점수" 팝업
- 팝업은 0.8초간 위로 떠오르며 페이드아웃
- 회의록 줄과 일반 줄 동시 클리어 시 각각 따로 계산 후 합산

#### 점수 시스템

##### 회의록 줄 (글자 있는 줄) — 고득점
| 이벤트 | 점수 |
|--------|------|
| 1줄 클리어 | 200 |
| 2줄 동시 클리어 | 500 |
| 3줄 동시 클리어 | 1000 |
| 4줄 동시 클리어 | 2000 |

##### 일반 줄 (글자 없는 줄) — 저득점
| 이벤트 | 점수 |
|--------|------|
| 1줄 클리어 | 50 |
| 2줄 동시 클리어 | 100 |
| 3줄 동시 클리어 | 200 |
| 4줄 동시 클리어 | 400 |

##### 기타
| 이벤트 | 점수 |
|--------|------|
| 게임 종료 시 남은시간 보너스 | timeLeft × 10 |

#### 난이도 커브 (10초 간격 단계적 상승)
| 경과 시간 | 라인 상승 간격 | 블록 낙하 속도 |
|-----------|---------------|---------------|
| 0~9초 | 7000ms | 600ms |
| 10~19초 | 6600ms | 560ms |
| 20~29초 | 6200ms | 520ms |
| 30~39초 | 5800ms | 480ms |
| 40~49초 | 5400ms | 440ms |
| 50~60초 | 5000ms | 400ms |
| (최솟값) | 3000ms | 250ms |

#### 게임 흐름
1. `create()` → HUD(점수·타이머), 12×24 그리드 초기화, 1줄 push, 첫 피스 스폰, 드래그 입력 세팅, 60초 타이머
2. `update()` → 낙하 타이머 체크(자동 하강/고정) + 라인 상승 타이머 + 10초 단위 난이도 갱신 + 매 프레임 그리드 렌더링
3. **게임 오버 조건**: 새 피스 스폰 시 충돌 / 라인 상승 시 0행 초과 / 60초 타이머 종료
4. 종료 후 0.5초 딜레이 → debug 모드면 BootScene, 아니면 ResultScene으로 전환

#### 비주얼
- 올라오는 블록: 회색 톤 (`#5a5a6e`) + 기획서 본문 한 글자 (회색 텍스트 `#cccccc`, 11px)
- 플레이어 블록: 표준 테트리스 7색
- 고스트 피스: 25% 투명도
- 클리어 팝업: 골드 (`#ffd700`), 검정 외곽선, 위로 떠오르며 1.5배 확대 후 사라짐

### STAGE 5. 퇴근 — 술 버리기 (슈의 맛동산 스타일)

> "막내야 회식가자! 술 빼기 없어! 네발로 기어갈 준비 됐지?"

- **장르**: 타이밍 액션 (슈의 맛동산 스타일)
- **컨셉**: 부장과 마주보고 회식 중! 부장이 딴짓할 때 술을 몰래 버려라!

#### 레이아웃
- **부장** (상단): 이미지 + 상태 텍스트 (정면/왼쪽/오른쪽/음주/주문/화남)
- **상태 표시** (중앙): "🚨 쳐다보는 중!" (빨강) / "✅ 지금이야! 버려!" (초록)
- **플레이어** (중앙 하단): 앉아있기 / 술 버리기 이미지
- **버리기 버튼** (하단): 큰 빨간 버튼 "🍺 꾹 눌러서 버려!"
- 술잔은 표시하지 않음 (항상 가득 차 있다고 간주)

#### 조작법
- **버튼 꾹 누르기**: 술 버리기 (초당 40 점수, 오래 누를수록 많이 버림)
- **버튼 떼기**: 버리기 중단

#### 부장 행동 패턴
- **안전 상태** (버려도 됨): 왼쪽 보기, 오른쪽 보기, 술 마시기, 주문 중
- **위험 상태** (버리면 안 됨): 정면 응시
- 1~3개 안전 상태 후 정면 전환 반복
- 시간 경과에 따라 난이도 증가: 안전 시간 감소, 위험 시간 증가

#### 걸림 판정
- 위험 상태(정면)에서 버튼 누르면 → 즉시 걸림
- 안전 상태에서 누르고 있다가 부장이 정면 전환 → 즉시 걸림
- 걸리면: 😡 화남 + 화면 흔들림 + **-30점**

#### 점수 시스템
| 이벤트 | 점수 |
|--------|------|
| 술 버리기 (초당) | +40 |
| 걸림 (들킴) | -30 |
- **점수** = 총 버린 양 (상한 없음)

## 게임 규칙

### 제한시간
- 각 게임의 제한시간은 **60초 (1분)**
- 60초 동안 최대한 많은 점수를 획득
- 시간 초과 시 해당 스테이지 획득 점수로 마감

### 점수 시스템
- **점수 상한 없음** (무제한)
- 빠르게 / 많이 할수록 높은 점수
- 스테이지별 고유 점수 산정 방식
- 5개 스테이지 총합으로 랭킹 경쟁

### 진행 방식
1. **출근** → **오전** → **점심** → **오후 회의** → **퇴근** 순서로 진행
2. 60초가 지나면 자동으로 다음 스테이지로 이동
3. 모든 스테이지 완료 후 최종 점수 집계
4. 점수에 따른 등급 및 랭킹 표시

## 스테이지 흐름

```
타이틀 화면
  ↓ [출근하기]
스테이지 인트로 (카운트다운 3-2-1-GO)
  ↓
미니게임 플레이 (60초)
  ↓
결과 화면 (점수 표시)
  ↓
다음 스테이지 인트로
  ↓
... (5회 반복)
  ↓
최종 결과 & 랭킹 화면
```

## 랭킹 시스템

### 등급
| 등급 | 점수 범위 | 이모지 | 칭호 |
|------|-----------|--------|------|
| S | 5,000+ | 🏆 | 퇴근의 달인 |
| A | 3,000~4,999 | 😊 | 프로 직장인 |
| B | 1,500~2,999 | 😐 | 평범한 회사원 |
| C | 500~1,499 | 😤 | 고군분투 사원 |
| D | 0~499 | 💀 | 사회초년생 |

### 랭킹 저장
- 서버 연동 랭킹 보드 (추후)
- 로컬 스토리지 기반 임시 랭킹
- 닉네임 + 총점 + 등급 표시
- 친구에게 랭킹 공유 기능
